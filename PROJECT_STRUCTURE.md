# Структура проекта

Проект использует **feature-first архитектуру** для лучшей организации кода.

```
src/
├── app/                    # Конфигурация приложения
│   ├── App.jsx            # Главный компонент с роутингом
│   └── index.css          # Глобальные стили
│
├── features/              # Основные фичи приложения
│   ├── auth/              # Аутентификация
│   │   ├── AuthContext.jsx    # Контекст аутентификации
│   │   ├── LoginPage.jsx      # Страница входа
│   │   └── index.js           # Экспорты
│   │
│   ├── qr-code/           # QR-код для подключения
│   │   ├── QRCodePage.jsx     # Страница с QR-кодом
│   │   └── index.js
│   │
│   ├── level-selection/   # Выбор уровня
│   │   ├── LevelSelectionPage.jsx
│   │   └── index.js
│   │
│   ├── level/             # Прохождение уровня
│   │   ├── LevelPage.jsx      # Страница уровня с диалогами
│   │   └── index.js
│   │
│   └── statistics/        # Статистика для учителя
│       ├── StatisticsPage.jsx
│       └── index.js
│
└── shared/                # Общие компоненты и утилиты
    ├── components/         # Переиспользуемые компоненты
    │   └── ProtectedRoute.jsx
    │
    ├── config/            # Конфигурация
    │   └── firebase.js        # Настройка Firebase
    │
    ├── hooks/             # Кастомные хуки
    │   └── useLocalIP.js      # Получение локального IP
    │
    └── utils/             # Утилиты
        └── storage.js          # Работа с localStorage
```

## Основные фичи

### 1. QR-код (`features/qr-code`)
- Генерирует QR-код с URL для подключения учеников
- Определяет локальный IP адрес для доступа с других устройств
- Отображает ссылку для ручного ввода

### 2. Аутентификация (`features/auth`)
- Анонимная аутентификация через Firebase
- Сохранение имени пользователя
- Защищенные маршруты

### 3. Выбор уровня (`features/level-selection`)
- Отображение доступных уровней
- Прогрессивное открытие уровней
- Сохранение прогресса

### 4. Прохождение уровня (`features/level`)
- Диалоги с персонажами
- Кнопка "Далее" для перехода между репликами
- Теоретический материал
- Автоматическое разблокирование следующего уровня

### 5. Статистика (`features/statistics`)
- Общая статистика по ученикам
- Прохождение по уровням
- Средний прогресс

## Хранение данных

Все данные хранятся в `localStorage`:
- Прогресс пользователя: `progress_{userId}`
- Прогресс уровня: `level_{levelId}_{userId}`
- Имя пользователя: `userName_{userId}`
